DIY RFLink, Transmit Method

10 August, 2018

15:46

 

Because the signal transmission is so straight forward, there are minor differences in functional encoding, start en stop sequences, it's best to give each protocol class it's own method, here called Home_Command. This method should return true if it has accepted and send the message. As an example below the EV1527 protocol (later the PT2262 will be added to this).

 

If we want to use another transmitter, e.g. SI4432, then we have to write a second method and choose between those 2 methodes in RFL_Protocols.h.
